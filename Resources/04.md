**Retail Trends Run Guide and QA Checklist v2**

*Standard operating procedure for repeating the same prompt with only three changing parameters*

This guide assumes the master prompt, front-section template, and workbook template are uploaded into one project and reused without structural edits.

---

# 1) What never changes

- Source hierarchy and prohibited-source list.
- Bottom-up method: pulses to Whisper Tags to Macro Trends.
- Thresholds for Whisper Tags and Macro Trends.
- Single-assignment rules for pulses, Whisper Tags, CIPHER buckets, and attribute-led buckets.
- Technology extraction discipline from pulses only.
- Front-section order, appendix schemas, and final quality gate.

# 2) What you edit each run

| **Field** | **Value to edit** |
| --- | --- |
| Country | [Country] |
| Timeline | [YYYY-MM-DD] to [YYYY-MM-DD] |
| Retailers in scope | [Retailer 1, Retailer 2, Retailer 3, ...] |

Do not edit any other part of the master prompt. Do not rename sheets or change column headers in the workbook.

# 3) Project setup

1. Create one project for the retail trends workflow.
2. Upload the four master assets: master prompt template, front-section template, appendix workbook template, and this run guide.
3. Pin or clearly name the assets as master sources so they are not confused with finished country outputs.
4. Add project instructions telling the model to use the uploaded master prompt as the governing methodology, the front-section file as the mandatory narrative structure, the workbook as the mandatory appendix schema, and the QA checklist as the validation gate.

# 4) Country-run sequence

1. Start a fresh chat for each country run.
2. Paste only the new parameter block: Country, Timeline, Retailers in Scope.
3. Build the retailer structure map first.
4. Collect and de-duplicate up to 100 pulses.
5. Code each pulse to 1-3 attributes using the attribute list and the change-scan definitions.
6. Cluster pulses into mechanism-based Whisper Tags using the stated thresholds and single-assignment rules.
7. Elevate accepted Whisper Tags into 6 primary Macro Trends using the macro thresholds and mechanism-only clustering rule.
8. Run the secondary CIPHER regrouping on accepted Whisper Tags without altering primary Macro assignments.
9. Run the secondary attribute-led regrouping on accepted Whisper Tags without altering primary Macro assignments.
10. Run the falsification check using credible counter-evidence within the same timeframe.
11. Run the separate pulse-only technology pass and build technology trends from explicit deployments only.
12. Complete the appendix workbook first.
13. Write the front section second, using the completed workbook only.

# 5) Workbook usage notes

- Run_Parameters is the only place where country-level inputs should be edited before the analysis begins.
- Method_Reference and Attribute_Library are locked reference sheets. They restate the original method so it does not drift between runs.
- Retailer_Structure_Map is the coverage-control sheet for banners, formats, owned brands, and subsidiaries.
- Pulse_Ledger, Whisper_Tags, Macro_Trend_Map, WhisperTag_CIPHER_Map, WhisperTag_Attribute_Map, Technology_Trend_Map, and Reference_Shortlist are the core appendix sheets.
- Falsification_Log records counter-evidence that narrows or drops weak tags or Macro Trends.
- Executive_Readout and QA_Check are control sheets that help turn the appendices into the final narrative and validation review.

# 6) QA checklist before accepting any country output

1. All sources are preferred English and inside the exact time window.
2. Each pulse is a unique real-world event and not a syndicated duplicate.
3. Each pulse is coded to 1-3 attributes only.
4. Each used Pulse_ID maps to one Primary_WhisperTag_ID.
5. Each used WhisperTag_ID maps to one primary Macro Trend or is explicitly marked Unassigned.
6. Whisper Tags and Macro Trends are mechanism-led and not generic category labels.
7. CIPHER and attribute-led regroupings are secondary views only and do not alter the primary assignments.
8. Technology trends come from a separate pulse-only pass and not from Whisper Tags.
9. Front section is written after the workbook is complete.
10. Every front-section sentence has Harvard-style in-text citations.
11. No em dashes, no speculation, no invented numbers, and no unsourced claims.

# 7) Common failure modes to block

- Starting with pre-written themes instead of pulse collection.
- Using Tier 3 to originate Macro Trends instead of validating them.
- Letting one pulse appear in more than one Whisper Tag or one proof point set.
- Grouping by adjacent function or technology instead of repeated mechanism.
- Creating technology trends from Whisper Tags instead of from pulses.
- Reusing a prior country workbook as the starting template instead of the blank master workbook.

# 8) Output packaging

- Return one Excel workbook containing the appendix sheets and helper/control sheets.
- Return one front-section readout following the fixed section order.
- Name outputs consistently, e.g., [Country]_Retail_Trends_Appendix_v1.xlsx and [Country]_Retail_Trends_Front_Section_v1.docx.
